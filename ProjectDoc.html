Salary Formula

Gross Salary = Base Pay + HRA (20% of Base) + DA (10% of Base)
Deductions   = PF (12% of Base) + Income Tax (10% of Gross)
Net Salary   = Gross Salary - Deductions

4. File-by-File Explanation

4.1 model/Employee.java

AspectDetail
What it isAbstract base class representing any employee in the system
What it doesStores common fields (empId, name, department, baseSalary), provides getters/setters, defines abstract getEmployeeType(), and toCSV() for file saving

public abstract String getEmployeeType();

4.2 model/FullTimeEmployee.java & PartTimeEmployee.java & ContractEmployee.java

AspectDetail
What they areConcrete subclasses of Employee
What they doCall super() constructor and implement getEmployeeType() returning "FULL_TIME", "PART_TIME", or "CONTRACT"
Java Concepts Inheritance (extends) Method Overriding Polymorphism super() constructor

4.3 manager/EmployeeManager.java

AspectDetail
What it isCentral manager class for all employee CRUD operations
What it doesAdd, get, update, delete, search employees. Loads data from CSV on start, saves on exit. Uses ArrayList for ordered storage and HashMap for fast lookup
How it worksaddEmployee() checks HashMap for duplicates → throws DuplicateEmployeeException. getEmployeeById() does O(1) map lookup → throws EmployeeNotFoundException if missing. searchByName() iterates ArrayList doing case-insensitive match
Java Concepts ArrayList HashMap Collections.synchronizedMap() Custom Exception Throwing Collections Framework 
private final List<Employee> employeeList = new ArrayList<>();
private final Map<String, Employee> employeeMap = Collections.synchronizedMap(new HashMap<>());

// O(1) lookup
public Employee getEmployeeById(String empId) throws EmployeeNotFoundException {
    Employee emp = employeeMap.get(empId);
    if (emp == null) throw new EmployeeNotFoundException(empId);
    return emp;
}

4.4 manager/PayrollManager.java

AspectDetail
What it isOrchestrates the multithreaded payroll run
What it doesCreates a fixed thread pool of 5 threads, submits one PayrollWorker per employee, waits for all to finish, then starts FileWriterThread for summary. Also provides department-wise payroll report
How it worksExecutorService.submit() queues each PayrollWorker. executor.shutdown() + awaitTermination(60s) blocks until all threads done. Then FileWriterThread.start() + .join() writes summary
Java Concepts ExecutorService Executors.newFixedThreadPool() Future awaitTermination() Thread Pool InterruptedException handling 
ExecutorService executor = Executors.newFixedThreadPool(5);
for (Employee emp : employees) {
    executor.submit(new PayrollWorker(emp));  // one thread per employee
}
executor.shutdown();
executor.awaitTermination(60, TimeUnit.SECONDS); // wait for all

4.5 thread/PayrollWorker.java

AspectDetail
What it isThe core thread task — runs salary calculation for one employee
What it doesImplements Runnable. In run(): sleeps 500ms (simulated delay), calculates Gross/Deductions/Net salary, writes to shared payroll_log.txt using synchronized, generates individual payslip, increments AtomicInteger counter
How it worksThread.sleep(500) simulates processing. synchronized writeToPayrollLog() ensures only one thread writes to the log file at a time — preventing data corruption
Java Concepts Runnable interface run() method Thread.sleep() synchronized method AtomicInteger InterruptedException FileWriter (append mode) 
public class PayrollWorker implements Runnable {
    @Override
    public void run() {
        Thread.sleep(500);                  // simulate delay
        // ... salary calculation ...
        writeToPayrollLog(...);             // synchronized write
        processedCount.incrementAndGet();   // AtomicInteger
    }

    private static synchronized void writeToPayrollLog(...) {
        // Only ONE thread enters at a time
    }
}

4.6 thread/FileWriterThread.java

AspectDetail
What it isA dedicated thread for writing the payroll summary footer
What it doesAfter all PayrollWorker threads finish, this thread appends total employees processed, total payroll cost, and run timestamp to payroll_log.txt
How it worksExtends Thread and overrides run(). PayrollManager calls .start() then .join() to wait for it
Java Concepts extends Thread run() override Thread.start() Thread.join() FileWriter

5. How to Compile & Run

Step 1 — Compile

cd PayrollSystem

javac -d out -sourcepath src ^
  src\com\payroll\Main.java ^
  src\com\payroll\model\*.java ^
  src\com\payroll\manager\*.java ^
  src\com\payroll\thread\*.java ^
  src\com\payroll\exception\*.java ^
  src\com\payroll\util\*.java ^
  src\com\payroll\db\*.java

Step 2 — Run

java -cp out com.payroll.Main

Or double-click compile.bat then run.bat.
